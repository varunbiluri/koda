🐛

How it works

Executes. Verifies. Self-corrects.

Koda isn't a chatbot that pastes code into your editor. It runs a full execute → verify → fix loop until the task is done or it hits a safety stop.

$ koda fix "users can't log in after password reset" ⚡ Koda — Autonomous Bug Fix users can't log in after password reset ── Step 1 / 3 ───────────────────── ○ Searching for password reset flow... ○ Found: src/auth/reset-service.ts ○ Root cause: token not invalidated after use ○ Patching src/auth/reset-service.ts ○ Verifying — running pnpm test... ✓ Fix verified in 1 step ✓ Done in 14.2s 3/4 tasks completed (75%) · ~1.2h saved
⚡

Execution, not suggestion

Koda writes the code, runs the tests, and gives you a verified diff. It doesn't paste suggestions into a chat window.

↩

Autonomous self-correction

If tests fail, Koda classifies the error, adjusts its approach, and tries again — up to 3 iterations by default.

◆

Learns from your repo

Every session, Koda records which strategies worked. Future fix attempts in the same repo are smarter.

⊕

Impact-aware changes

Before touching a high-dependency file, Koda shows you how many files it affects. You choose to proceed or not.

Safety model

Autonomous. Not reckless.

Koda has a three-tier permission system. It never runs a destructive operation without your explicit approval — even in autonomous mode.

✅

Read-only auto-approved

read_file, search_code, git_log — never needs your permission. Zero friction for analysis.

🔒

Writes require approval

write_file, git_commit, run_terminal — prompts before every state-changing operation.

🚫

Dangerous ops blocked forever

rm -rf, git push --force, DROP TABLE — unconditionally blocked. Cannot be overridden.

🧪

Verified before done

Koda considers a task complete only after pnpm build and pnpm test both pass. No silent breakage.

📊

Confidence scoring

Koda scores its own confidence (HIGH / MEDIUM / LOW). Low confidence triggers a stop and asks for guidance — not a guess.

💾

Your data stays local

All code, index files, and memory are stored in .koda/ in your repo. Nothing leaves your machine without your API key.

Quickstart

From install to first fix in under 5 minutes

Works with Node.js, Python, Go, and Rust projects.

1

Install Koda

npm install -g @varunbilluri/koda

Requires Node.js 18+. No other dependencies.

2

Index your repo

koda init

Koda reads your codebase, detects your stack, and shows you what it found. Takes ~10s for most projects.

3

Configure your AI provider

koda login

Supports Azure AI, OpenAI, Anthropic, and Ollama. Your key stays in .koda/config.json.

4

Fix your first bug

koda fix "describe the bug"

Watch Koda find, patch, and verify the fix end-to-end.
